# Roadmap

## Current stage

Openboard is in the internal MVP / operations-tool stage.

## Already built

- Dashboard MVP
- Tasks view
- Todos view
- Todo → Task conversion
- Dev Log groundwork
- Top navbar navigation
- Project selector support
- Mobile hamburger navigation

## Current priorities

1. Stabilize navigation and project switching
2. Review and merge feature branches cleanly
3. Improve mobile usability
4. Make Dev Log truly useful as project memory
5. Strengthen project-level organization

## Next likely features

- Project badges on tasks
- Project edit/delete
- Better Dev Log integration with completed tasks
- GitHub integration (issues / PRs / CI)
- Better filtering and search
- Notes / documentation links from within the app

## Open questions

- Should Openboard remain purely internal?
- How far should project documentation live inside the app vs repo docs?
- Which Biketime repos should be connected first?
