# Project Home

This is the source of truth for Openboard project context.

## What lives here

- **Product** — what Openboard is, who it serves, where it is going
- **Architecture** — how the system is put together
- **Decisions** — important choices and why they were made
- **Process** — how we work on the project

## Structure

- `product/vision.md` — product purpose, users, and principles
- `product/roadmap.md` — current priorities and next milestones
- `architecture/overview.md` — technical overview of the system
- `decisions/README.md` — decision log index + ADR format
- `process/workflow.md` — how Adam, TARS, Erik, and Brock work together

## Rules

1. Durable decisions belong here, not only in chat.
2. If a decision affects code or product direction, document it.
3. Keep docs short, clear, and updated.
4. Prefer editing existing docs over creating scattered new files.
