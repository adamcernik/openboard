# Decisions Log

Use this folder for important project decisions.

## When to add a decision

Create a decision note when a choice:
- affects product direction
- changes architecture or data model
- influences future implementation choices
- would be annoying to rediscover later

## Naming

Use numbered files like:
- `001-openboard-as-internal-hq.md`
- `002-todo-to-task-conversion.md`

## Suggested template

```md
# 001 Decision Title

## Status
Accepted

## Context
What problem are we solving?

## Decision
What did we choose?

## Consequences
What becomes easier / harder because of this?
```

## Initial decisions to document

- Openboard as the internal HQ for operations and development
- Todos are lightweight notes; Tasks are structured execution items
- Dev Log is for project memory and work history
- Repo docs are the durable source of truth for project knowledge
