# 001 Openboard as Internal HQ

## Status
Accepted

## Context
We need one lightweight internal place to manage work, keep context, and support collaboration between Adam and AI agents.

## Decision
Openboard will serve as the internal HQ for Openboard/Biketime-related operational work:
- task management
- todo capture
- dev log visibility
- project organization

Long-form documentation and durable decisions will live in repo docs, not only inside the app.

## Consequences
### Positive
- One practical center for daily execution
- Better visibility into active work
- A place where Adam, TARS, Erik, and Brock can align

### Tradeoffs
- We must avoid turning Openboard into a bloated all-in-one app
- Durable documentation still requires discipline in `docs/`
